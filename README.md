# Dungeondraft Tile Browser

Модуль для Foundry VTT, который импортирует объектные ассеты из Dungeondraft и превращает их в удобную библиотеку тайлов внутри мира Foundry.

Основной сценарий: у мастера есть `.dungeondraft_pack` или папка с картинками объектов, и нужно быстро находить нужный объект, фильтровать ассеты, перекрашивать поддерживаемые элементы и размещать их на сцене как Foundry Tiles.

## Требования

- Foundry VTT: версия 13.
- Минимальная версия Foundry: 13.
- Проверенная версия Foundry: 13.
- Права GM нужны для импорта паков, изменения общей библиотеки, удаления ассетов и очистки данных.

Совместимость указана в `dungeondraft-tile-browser/module.json`.

## Что умеет модуль

- Импортирует `.dungeondraft_pack` файлы.
- Достает из Dungeondraft-паков объектные текстуры из `textures/objects`.
- Импортирует обычные папки с изображениями.
- Поддерживает изображения `apng`, `avif`, `bmp`, `gif`, `jpg`, `jpeg`, `png`, `svg`, `tiff`, `webp`.
- Создает библиотеку паков и ассетов внутри мира Foundry.
- Добавляет кнопку открытия библиотеки в инструменты тайлов на сцене.
- Позволяет искать ассеты по названию, автору и тегам.
- Позволяет включать и отключать отдельные паки.
- Позволяет скрывать паки из библиотеки.
- Позволяет редактировать название и автора пака или ассета.
- Позволяет удалять отдельные ассеты или целые паки из индекса библиотеки.
- Поддерживает пользовательские теги для паков и ассетов.
- Автоматически импортирует теги из `default.dungeondraft_tags`.
- Помечает ассеты с тегом `colorable` как перекрашиваемые.
- Генерирует цветовые варианты для перекрашиваемых ассетов.
- Запоминает недавние и избранные цвета.
- Позволяет выбрать цвет через системный EyeDropper, если браузер Foundry его поддерживает.
- Размещает выбранный ассет на сцене как обычный Foundry Tile.
- Поддерживает предпросмотр тайла перед размещением.
- Поддерживает привязку к сетке, поворот, масштабирование и блокировку размещенного тайла.
- Запоминает последний использованный размер для каждого ассета.
- Может перестроить библиотеку по уже загруженным файлам в папке импорта.
- Может удалить тайлы, созданные этим модулем, со сцен текущего мира.
- Может очистить библиотеку и попытаться удалить файлы ассетов из папки импорта.

## Для чего применяется

Модуль нужен, когда карты в Foundry собираются или дорабатываются прямо на сцене:

- быстро разложить мебель, декор, дороги, растения, обломки, руины и другие объекты;
- использовать ассеты из Dungeondraft без ручного распаковывания каждого пака;
- держать несколько паков ассетов в одной поисковой библиотеке;
- импортировать собственные папки с PNG/WebP/SVG тайлами;
- перекрашивать цветовые варианты объектов, если пак размечен тегом `colorable`;
- не хранить каждый объект вручную в Journal или File Picker.

## Установка

Папка модуля должна называться:

```text
dungeondraft-tile-browser
```

Ее нужно поместить в папку модулей Foundry:

```text
FoundryVTT/Data/modules/dungeondraft-tile-browser
```

После этого модуль включается в настройках нужного мира Foundry.

## Как открыть библиотеку

После включения модуля открой сцену и перейди в инструменты тайлов. Модуль добавляет туда кнопку библиотеки Dungeondraft Tile Browser.

Также настройки модуля доступны через меню настроек модуля в Foundry.

## Импорт `.dungeondraft_pack`

Импорт паков доступен только GM.

При импорте модуль:

1. Читает Godot/Dungeondraft pack.
2. Находит папку `res://packs/...`.
3. Читает `pack.json`.
4. Читает `data/default.dungeondraft_tags`, если файл есть.
5. Берет только объектные текстуры из `textures/objects`.
6. Загружает найденные изображения в папку импорта Foundry.
7. Создает записи паков и ассетов в библиотеке.

Содержимое паков не хранится внутри папки модуля. Модуль копирует нужные текстуры в Data-хранилище Foundry через `FilePicker`.

## Импорт папки с изображениями

Кроме `.dungeondraft_pack`, модуль умеет импортировать обычную папку с изображениями.

При таком импорте:

- структура папок используется для группировки;
- имена директорий становятся тегами ассетов;
- автор пака указывается как `Folder Import`;
- ассеты из папки не считаются `colorable`, потому что у них нет Dungeondraft-разметки `default.dungeondraft_tags`.

## Где хранятся тайлы и данные

По умолчанию импортированные ассеты хранятся в Foundry Data по пути:

```text
assets/dungeondraft-tile-browser
```

Путь можно изменить в настройках модуля. Настройка называется import path.

Внутри папки импорта модуль создает примерно такую структуру:

```text
assets/dungeondraft-tile-browser/
  library-index.json
  pack-id/
    preview.png
    raw/
      pack.json
      default.dungeondraft_tags.json
    textures/
      objects/
        ...
    generated/
      ...
```

Для импортированных папок вместо `pack.json` создается:

```text
raw/folder-import.json
```

`textures/objects` содержит загруженные изображения, которые затем используются тайлами на сценах.

`generated` содержит созданные модулем цветовые варианты перекрашиваемых ассетов.

`library-index.json` хранит общий индекс библиотеки. Он нужен для синхронизации и восстановления библиотеки из файлов.

Кроме файла `library-index.json`, данные библиотеки также хранятся в настройках мира Foundry (`game.settings`) в ключе модуля `dungeondraft-tile-browser.library`.

## Как модуль размещает тайлы

Когда пользователь выбирает ассет в библиотеке, модуль запускает режим размещения:

- появляется полупрозрачный предпросмотр;
- клик левой кнопкой мыши размещает тайл;
- колесо мыши поворачивает тайл;
- `Shift + колесо` меняет масштаб;
- `Ctrl` позволяет разместить несколько копий подряд;
- `Esc` или короткий клик правой кнопкой отменяет размещение;
- зажатая правая кнопка позволяет панорамировать сцену штатным способом Foundry.

Размещенный объект создается как обычный Foundry `Tile`.

В flags тайла модуль сохраняет служебные данные:

- id ассета;
- id пака;
- выбранный цвет;
- DPI карты;
- состояние привязки к сетке;
- состояние блокировки;
- исходный путь к ассету.

Это позволяет модулю позже находить тайлы, созданные им, например для удаления тайлов модуля со сцен.

## Размеры, DPI и сетка

По умолчанию модуль считает, что карта использует `256 DPI`.

Если у ассета в имени есть размер вида `2x3`, модуль использует его как размер в клетках. Например, объект с `2x3` в имени будет размещаться как 2 клетки на 3 клетки.

Если размер в клетках не найден, модуль берет пиксельный размер изображения и пересчитывает его через DPI:

```text
размер на сцене = размер изображения * размер клетки Foundry / DPI
```

В интерфейсе библиотеки можно изменить DPI, включить или отключить привязку к сетке и включить блокировку размещаемых тайлов.

## Цветовые варианты

Цветовой инструмент появляется, когда среди видимых ассетов есть `colorable`-ассеты.

Для `.dungeondraft_pack` модуль определяет такие ассеты по тегу `colorable` или по custom color overrides из `default.dungeondraft_tags`.

При выборе цвета модуль не меняет оригинальный файл. Он создает отдельный вариант изображения в папке:

```text
assets/dungeondraft-tile-browser/pack-id/generated
```

Ссылка на сгенерированный вариант сохраняется в индексе библиотеки.

## Настройки

Модуль хранит часть настроек на уровне мира, часть на уровне клиента.

Мировые настройки:

- библиотека паков и ассетов;
- путь импорта;
- общий индекс библиотеки.

Клиентские настройки:

- язык интерфейса;
- размер карточек ассетов;
- шаг поворота;
- шаг масштабирования;
- привязка к сетке;
- блокировка размещаемых тайлов;
- DPI карты;
- запомненные размеры ассетов;
- палитра цветов.

## Очистка и восстановление

В настройках модуля есть служебные действия:

- перестроить библиотеку из файлов в папке импорта;
- удалить тайлы, созданные этим модулем, со всех сцен текущего мира;
- очистить данные модуля и попытаться удалить импортированные файлы.

Удаление файлов зависит от возможностей FilePicker API в текущей версии Foundry и используемого хранилища. Если Foundry не даст удалить файлы программно, модуль очистит библиотеку, но сами файлы может потребоваться удалить вручную.

Для защиты от случайного удаления модуль отказывается чистить слишком широкие пути вроде `assets`, `worlds` или `modules`. В пути импорта должен присутствовать идентификатор модуля `dungeondraft-tile-browser`.

## Ограничения

- Модуль импортирует из `.dungeondraft_pack` только объектные текстуры `textures/objects`.
- Модуль не импортирует стены, источники света, полноценные карты Dungeondraft или сцены.
- Импорт, удаление и редактирование общей библиотеки доступны только GM.
- Перекрашивание работает только для ассетов, которые модуль распознал как `colorable`.
- Уже размещенные тайлы ссылаются на файлы в папке импорта. Если удалить импортированные файлы, такие тайлы потеряют текстуры.
